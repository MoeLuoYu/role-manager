# Role Manager（角色管理）

一个功能完善的 WordPress 角色管理插件，提供直观的界面来管理站点所有角色及其权限。

## 功能特性

- **角色管理** — 查看、创建、删除自定义角色
- **角色改名** — 修改任意角色的显示名称，支持一键重置
- **权限编辑** — 精细化管理自定义角色的各项权限（Capability）
- **角色继承** — 支持设置父角色，子角色自动继承父角色的全部权限
- **受限权限** — 可配置受限权限列表，防止通过本插件分配高危权限
- **操作日志** — 记录所有角色管理操作，包含操作者、时间、IP 及详情
- **多语言支持** — 完整的 i18n 国际化适配，支持翻译为任意语言

## 安装

1. 将 `role-manager` 文件夹上传至 `/wp-content/plugins/` 目录
2. 在 WordPress 后台「插件」页面激活「角色管理」
3. 后台左侧菜单会出现「角色」入口

## 使用说明

### 角色管理页面

进入后台 **角色 → 角色管理**：

- 查看站点所有角色列表（标识、显示名称、类型、用户数）
- 点击「改名」修改角色显示名称
- 点击「编辑权限」管理自定义角色的权限（WordPress 默认角色不可编辑权限）
- 点击「删除」删除自定义角色（需输入角色标识确认，WordPress 默认角色不可删除）

### 创建新角色

在角色管理页面右侧的「创建新角色」面板中：

1. 填写**角色标识**（仅限小写字母、数字和下划线）
2. 填写**显示名称**
3. 可选择**父角色**（新角色将自动继承父角色的所有权限）
4. 点击「创建角色」

### 角色继承

- 创建角色时可指定父角色，子角色会继承父角色的全部权限
- 父角色的权限变更会自动向下级联更新到所有子角色
- 支持多层继承，并内置循环继承检测

### 受限权限

进入后台 **角色 → 设置**：

- 受限权限无法通过本插件分配给任何角色
- 默认限制包括 `manage_options`、`edit_plugins`、`install_plugins` 等高危权限
- 可根据需要自定义受限权限列表

### 操作日志

进入后台 **角色 → 操作日志**：

- 查看创建、删除、改名、重置、权限更新等操作的完整记录
- 记录包含操作时间、操作者、操作类型、详情和 IP 地址
- 支持一键清空日志

## 多语言翻译

插件已完整适配 WordPress i18n 体系。添加新语言翻译：

1. 复制 `languages/role-manager.pot` 并重命名为 `role-manager-{语言代码}.po`（如 `role-manager-ja.po`）
2. 使用 [Poedit](https://poedit.net/) 打开 `.po` 文件完成翻译
3. 保存后 Poedit 会自动生成对应的 `.mo` 文件
4. 将 `.po` 和 `.mo` 文件放入 `languages/` 目录即可
