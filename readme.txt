一个功能完善的 WordPress 角色管理插件，提供直观的界面来管理站点所有角色及其权限。

== Description ==

**角色管理 (Role Manager)** 为 WordPress 站点管理员提供了一套完整的角色管理解决方案，让你可以轻松管理站点中的所有角色。

**主要功能：**

* **角色管理** — 查看、创建、删除自定义角色
* **角色改名** — 修改任意角色的显示名称，支持一键重置
* **权限编辑** — 精细化管理自定义角色的各项权限（Capability）
* **角色继承** — 支持设置父角色，子角色自动继承父角色的全部权限，并内置循环继承检测
* **受限权限** — 可配置受限权限列表，防止通过本插件分配高危权限（如 `manage_options`、`edit_plugins` 等）
* **操作日志** — 记录所有角色管理操作，包含操作者、时间、IP 地址及操作详情
* **多语言支持** — 完整的 i18n 国际化适配，可翻译为任意语言

**安全特性：**

* 所有 AJAX 请求均验证 Nonce 和用户权限
* 所有输入数据经过严格的 sanitize 处理
* 受限权限机制防止非管理员分配高危权限
* WordPress 默认角色的核心权限不可被修改或删除
* 操作日志可审计追溯

= 使用说明 =

1. 激活插件后，后台左侧菜单会出现「角色」入口
2. 进入 **角色 → 角色管理** 页面查看和管理所有角色
3. 进入 **角色 → 操作日志** 页面查看操作历史
4. 进入 **角色 → 设置** 页面配置受限权限

== Installation ==

1. 将 `role-manager` 文件夹上传至 `/wp-content/plugins/` 目录，或在后台「插件」页面搜索安装
2. 在 WordPress 后台「插件」页面激活「角色管理」
3. 后台左侧菜单会出现「角色」入口

== Frequently Asked Questions ==

= 可以修改 WordPress 默认角色的权限吗？ =

不可以。出于安全考虑，WordPress 默认角色（管理员、编辑、作者、投稿者、订阅者）的权限不可被修改或删除，但可以修改它们的显示名称。

= 删除角色后，原来属于该角色的用户会怎样？ =

删除角色后，该角色的所有用户会自动被转移到 WordPress 站点设置的默认角色（通常为订阅者）中。

= 什么是角色继承？ =

创建新角色时可以指定一个父角色，新角色将自动继承父角色的全部权限。当父角色的权限发生变更时，子角色的继承权限也会自动更新。支持多层继承。

= 什么是受限权限？ =

受限权限是指被标记为不可通过本插件分配的权限，通常为高危权限（如管理站点选项、编辑插件等）。你可以在「设置」页面自定义受限权限列表。
